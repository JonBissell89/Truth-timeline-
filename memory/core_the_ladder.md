# The ladder of consolidation (founder + assistant, this conversation)

Meaning is built in rungs. Each rung is a DIFFERENT operation, not a smaller
pile. The six operations are the named verbs of the product:

```
thought → speech → memory → definition → meaning → understanding
 utter     ground   record   define       derive    resolve
```

- **thought** — private, lossy, no shared referent. Can't enter a system.
- **speech (ground)** — the first grounding act: two minds point at the same
  referent. Meaning is born HERE — in the shared pointing — nowhere earlier.
  (This is why the target conversation is the spec, not decoration.)
- **memory (record)** — the referent survives the moment. Raw, unindexed.
- **definition (define)** — Field X = Definition Y. First COMPRESSION and
  first IRREVERSIBILITY: throw away phrasing, keep referent. A field is a
  frozen agreement about what a thing IS, stable across conversations.
- **meaning (derive)** — definitions strung together = an ATTRIBUTE. KEY
  INSIGHT: meaning is DERIVED, not stored. "Survey Completed + Install
  Started" *means* "the handoff happened" — the meaning is in the RELATION,
  not in either field. **A field is a remembered fact; an attribute is a
  computed meaning. Meaning is the verb that runs over remembered facts.**
- **understanding (resolve)** — enough meanings that who/what/where/when/why
  collapse to ONE reading with no competing interpretation. (= grounded
  certainty, not felt confidence.)

# The compression (the real claim)

```
10,000,000,000 conversations
        ↓  ground:  pointing → referent
   100,000 fields           (100,000× )
        ↓  derive:  fields → meaning
       100 attributes       (1,000× )
        ↓  resolve: meanings → process reading
        10 processes        (10× )
```

A billion-to-one. The claim isn't "say less" — it's that the hierarchy lets
you **say one thing and have it unfold into the right ten thousand.** "Refine
the lead process" touches process 3 → its ~10 attributes → their ~1000
fields. The single-hop walk. Compression is what makes the walk sufficient.

# It's bidirectional — so it's a LANGUAGE, not a ladder

- **Up** = learning (conversations consolidate into understanding).
- **Down** = speaking (one understanding unfolds into the right thousand).

A structure climbed both ways, where each rung is a stable agreement, is a
**language a business/person can speak about itself** — vocabulary = fields,
grammar = attributes, a sentence = a process. And (the law) every word can
walk back DOWN to the conversations that defined it.

# Orenda builds the BOTTOM of this ladder

Heron's fields/attributes are the CONSOLIDATED (top) end. Orenda builds the
RAW, ACCRETING (bottom) end — utterances → nodes → strength — where truth
first lands. A node, once strong enough, is what a Field/Attribute later gets
PROPOSED from. Built so it CAN connect upward, not as an island.
