# What Orenda is

**Orenda is a visual representation of truth.** You speak — to a Claude
agent, on your phone, on your laptop, in your ear while mountain biking.
Out of the ten million words you say, meaning consolidates. Orenda is where
that meaning becomes **visible**.

- **The agent is the voice** — Claude, conversational, foundational,
  everywhere. This already exists; it is the part Anthropic built.
- **Orenda is the body** — the visible residue of the conversation. THIS is
  the part the founder builds. "Visually this is where I step in."
- It is **one product, two faces**: the talking (the agent's) and the seeing
  (the founder's). Same object, two organs.

*Orenda* (Iroquois): the spiritual force a person exerts to change the
world. The map is built from the user's own words, accreting over time.

# It is a tool for BOTH the human AND the agent

This is load-bearing and easy to miss. The map is not just a picture for the
human. It is **shared cognition**:

- The human **sees** it (rendered, glanceable, eventually biofabric).
- The agent (Claude) **traverses** it. "Recall that thing I said about X" →
  the agent finds the node and walks the rope DOWN to the exact utterances
  that grounded it. The origin is always recoverable.

The human navigates by eye; the agent navigates by traversal. Same graph.

# Voice-first, multi-device

Primary input is **speech while moving** — hands-free, eyes-off, mid-trail.
An utterance is a transcribed spoken fragment, possibly half-formed. The map
must be glanceable, not read. Phone + laptop show the SAME map (one person,
one truth, synced — so the store is server-side from the moment sync matters).

# Relationship to Heron

Orenda is a **separate product** from Heron, in its own repo
(`Desktop/Orenda`, peer to Heron, NEVER inside it). The CONNECTION: the
last hours of Heron work climbed the abstraction ladder (workflow → ontology
→ personal truth-map) and discovered the ENGINE underneath. Orenda takes that
engine to its own home. **Nothing copies over but the ideas** — the
no-hardcode discipline and the moat travel as principles, not as code.

The founder "started this years ago but felt it was not something I could
build." Why it's buildable now: the two walls that kill truth-graph projects
— the *resolution wall* (when are two utterances the same node?) and the
*blank-map wall* (nothing fills it) — were both dissolved by REASONING, not
engineering. Resolution is deferred (manual pointing is honest for now). The
blank-map wall is gone because **the conversation IS the fill mechanism**.

# The future shape (build coworker first)

Eventually: many agent "profiles" — a therapist, a coworker, a workout
buddy — each a lens on the same map structure. **Build the COWORKER lens
first.** The substrate is profile-agnostic by design.

# Aesthetic

Dark, deep, calm, clean — "Claude traveling in space." A traveler's
instrument panel: quiet, trustworthy, future-but-warm. NOT neon, NOT cold
sci-fi. Nodes feel like distant lights you navigate toward. Set tokens for
this from the start so it never has to be retrofitted.
